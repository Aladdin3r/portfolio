Overview

SafeCycle is a mobile map application catered toward cyclists and pedestrians with the goal of creating routes to destinations that are as safe as possible and can accommodate various mobility handicaps (ex: wheelchair accessibility).

Role: Project manager, UX Designer, Lead Developer

Duration: 4 months

Team:

  • Nea Pieroelie
  • Andrew Turitsa

Tech Stack:

  • Figma
  • NextJS
  • Google Maps API

Context

SafeCycle was created in response to the increasing number of pedestrian and cyclist accidents in urban areas. The app was designed to help users find the safest route to their destination by allowing users to input warnings for high-traffic areas, construction zones, and safety hazards.

Since pedestrians include a wide-range of differently-abled people, SafeCycle had to have personalized navigation options for accessibility.

Plan

To ensure that SafeCycle achieves its goal of providing safe routes for pedestrians and cyclists, the team decided to implement the following features:

  • Customizable route options
  • Real-time traffic/safety updates (by user input)
  • Knowledge of transportation safety through a quiz

Action

We started off wireframing the main features of the app, jumping straight to hi-fi prototypes to test with users. Some initial issues pointed out through user testing were:

  • Difficulty navigating to the home page (Severe)
  • Difficulty in finding the accessibility options with saved routes (moderate)

Having tackled these design issues, we moved on to developing the app using NextJS and Google Maps API. This is where many issues occurred, as this was the first time implementing an API into a project, and the API itself had many limitations.

Results

SafeCycle was able to provide a safe route for pedestrians and cyclists, but the app was not as user-friendly as we had hoped. The app was able to provide routes that avoided high-traffic areas and construction zones, but the app was not able to provide accessibility options for differently-abled users.

The app was also not able to provide real-time traffic updates, as the Google Maps API was not able to provide this feature.
